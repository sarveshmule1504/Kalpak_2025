
# Kalpak 2025

A responsive and interactive web application for **Kalpak 2025**. This platform serves as the central hub for event information, leadership profiles, and attendee registration.

## 🌟 Features
* **Event Management:** Detailed schedules and information about upcoming events (`events.html`).
* **Registration Portal:** Streamlined user registration for participants (`registration.html`).
* **Leadership Showcase:** Profiles highlighting the organizing committee and leadership team (`leadership.html`).
* **Interactive UI:** Dynamic elements powered by JavaScript for a smooth user experience.
* **Responsive Design:** Custom styling to ensure the site looks great on both desktop and mobile devices.

## 🛠️ Tech Stack
* **Markup:** HTML5
* **Styling:** CSS3 (`style-1.css`)
* **Scripting:** Vanilla JavaScript (`script-1.js`)

## 📂 Project Structure
* `index-2.html`: The main landing/home page of the website.
* `about.html`: Information regarding the purpose and history of Kalpak.
* `events.html`: Displays the schedule and details for Kalpak 2025 events.
* `leadership.html`: Showcases the team behind the event.
* `registration.html`: The sign-up and registration form for attendees.
* `contact.html`: Contact information and inquiry forms.
* `style-1.css`: Global stylesheet containing the site's design system.
* `script-1.js`: Main JavaScript file handling interactivity and form validations.
* `kalpak/`: Directory containing additional project assets.

## 💻 Local Setup & Installation

Since this is a static website, setting it up locally is incredibly simple.

1. **Clone the repository:**
   ```bash
   git clone [https://github.com/sarveshmule1504/Kalpak_2025.git](https://github.com/sarveshmule1504/Kalpak_2025.git)
   cd Kalpak_2025

```

2. **Run the project:**
* **Option A (Quickest):** Simply double-click on the `index-2.html` file to open it in your default web browser.
* **Option B (Recommended for Dev):** Use an extension like **Live Server** in VS Code. Right-click `index-2.html` and select "Open with Live Server" to enable hot-reloading as you make changes.
